# ⚡ 快速启动指南

## 问题分析

根据您的输出，发现了几个问题：

1. **只读模式**：私钥环境变量没有被正确读取
2. **自动跟单已禁用**：虽然配置了 `autoFollow: true`，但程序显示已禁用
3. **正在监听 0 个钱包**：过滤条件太严格或钱包信息获取失败

## 解决方案

### 方案 1：使用 PowerShell 脚本启动（推荐）

我已经创建了 `start.ps1` 脚本，可以直接使用：

```powershell
# 在 PowerShell 中运行
.\start.ps1
```

这个脚本会：
- 自动设置环境变量
- 显示配置状态
- 启动程序

### 方案 2：在同一 PowerShell 会话中运行

确保在同一个 PowerShell 窗口中设置环境变量和运行：

```powershell
# 1. 设置环境变量
$env:POLYMARKET_PRIVATE_KEY="0xd4ae880287b31d8316f31e938a4bb50d6260d765229076be83d8fa7962f2531b"

# 2. 验证设置
echo $env:POLYMARKET_PRIVATE_KEY

# 3. 在同一窗口中运行
npm start
```

### 方案 3：直接在 config.js 中填入私钥（快速但不推荐）

如果环境变量一直有问题，可以临时直接在 `config.js` 中填入：

```javascript
privateKey: '0xd4ae880287b31d8316f31e938a4bb50d6260d765229076be83d8fa7962f2531b',
```

⚠️ **注意**：这样做不够安全，仅用于测试。

### 方案 4：使用 cross-env（需要安装）

```bash
# 安装 cross-env
npm install -g cross-env

# 运行
cross-env POLYMARKET_PRIVATE_KEY=0xd4ae880287b31d8316f31e938a4bb50d6260d765229076be83d8fa7962f2531b npm start
```
