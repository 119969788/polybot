# 🚀 快速代理配置指南

## 方法一：使用启动脚本（最简单，推荐）

我已经创建了代理启动脚本，直接运行：

```powershell
.\start-with-proxy.ps1
```

脚本会自动：
- ✅ 设置代理环境变量
- ✅ 测试代理连接
- ✅ 启动程序

### 修改代理地址

如果您的代理端口不是 7890，编辑 `start-with-proxy.ps1`，修改：

```powershell
$PROXY_PORT = "7890"  # 改为您的实际端口
```

## 方法二：手动设置代理（PowerShell）

```powershell
# 设置代理（Clash 默认端口 7890）
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
$env:http_proxy="http://127.0.0.1:7890"
$env:https_proxy="http://127.0.0.1:7890"

# 验证设置
echo "HTTP_PROXY: $env:HTTP_PROXY"
echo "HTTPS_PROXY: $env:HTTPS_PROXY"

# 测试连接
curl -x http://127.0.0.1:7890 https://polymarket.com

# 如果测试成功，运行程序
npm start
```

## 查找您的代理端口

### Clash

1. 打开 Clash 客户端
2. 点击"设置"或"Settings"
3. 查看"端口"或"Port"设置
4. HTTP 代理端口通常是 **7890**

### 其他代理软件

查看您代理软件的配置界面，找到 HTTP 代理端口。

## 常见代理端口

| 代理软件 | HTTP 端口 | SOCKS5 端口 |
|---------|----------|------------|
| Clash | 7890 | 7890 或 7891 |
| Clash Verge | 7890 | 7890 或 7891 |
| V2Ray | 10809 | 10808 |
| Shadowsocks | - | 1080 |
| 系统代理 | 8080 | - |

## 测试代理是否工作

运行测试脚本：

```powershell
# 设置代理
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"

# 测试连接
Invoke-WebRequest -Uri "https://polymarket.com" -Proxy "http://127.0.0.1:7890" -UseBasicParsing
```

如果返回状态码 200，说明代理工作正常。

## 如果不知道代理端口

可以尝试：

```powershell
# 测试常见端口
$ports = @(7890, 10809, 1080, 8080, 7891)

foreach ($port in $ports) {
    try {
        $result = Invoke-WebRequest -Uri "https://polymarket.com" -Proxy "http://127.0.0.1:$port" -TimeoutSec 3 -UseBasicParsing -ErrorAction Stop
        Write-Host "✅ 端口 $port 可用！(状态码: $($result.StatusCode))" -ForegroundColor Green
        break
    } catch {
        Write-Host "❌ 端口 $port 不可用" -ForegroundColor Gray
    }
}
```
